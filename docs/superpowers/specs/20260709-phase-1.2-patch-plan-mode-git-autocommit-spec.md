## 2. 设计

### 2.1 Patch

新增行为：

1. **空 search 拒绝** → `PATCH_EMPTY_SEARCH`  
2. **全部应用后内容未变** → `PATCH_NOOP`（不写盘）  
3. **`dry_run: true`** → 校验并返回 unified diff，**不写盘**  
4. **成功摘要** → `chunks=N +added/-removed`  
5. **文件过大**（默认 2MiB）→ `PATCH_FILE_TOO_LARGE`，避免 LCS OOM  

### 2.2 Plan Mode

- 会话态：`agent`（默认）| `plan`  
- Plan 下优先 deny：`write` / `patch` / `bash` / `subtask` / `browser_fetch`  
- 只读仍可用：`read` / `search` / `todo` / `planner` / `skill` / `url_fetch` 等  
- 实现：`HookManager.setPlanMode(bool)` 在权限规则前叠加 plan deny 规则  
- `/plan on|off|status`；`/plan <desc>` 进入 plan 并排队计划 prompt  
- 兼容旧文案：明确说明「这是轻灵 Plan Mode，会禁写工具」

### 2.3 Git Auto-Commit

- Env：`QLING_GIT_AUTO_COMMIT` ∈ `off|on|ask`（默认 `off`）  
- 触发：工具 `write` / `patch` 成功且非 dry_run  
- `on`：`git add -- <path>` + `git commit -m "qling: update <relpath>"`（仅当仓库且有变更）  
- `ask`：返回提示，不自动 commit（会话侧可后续审批扩展）；本阶段输出明确提示用户 `/commit`  
- 非 git 仓库 / git 失败：warn 日志，不把工具结果改成 error
