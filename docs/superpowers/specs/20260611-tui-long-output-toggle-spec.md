# TUI Long Output Toggle Spec

## 背景

当前 TUI 会自动折叠超过 12 行的工具输出，并显示 `(Ctrl+O to expand)`，但 `Ctrl+O` 处理分支实际为空操作。这个提示与行为不一致，会破坏 Claude Code 式交互的可信度。

## 目标

- `Ctrl+O` 在本地 TUI 中切换“后续长工具输出”的展开/折叠模式。
- 默认保持折叠，避免长日志淹没终端历史。
- 展开模式只影响之后打印的工具输出，不尝试重写已经 append 到终端历史的旧输出。
- 切换时给出本地提示，不提交输入、不丢弃正在编辑的 prompt。
- `/shortcuts` 帮助同步说明真实行为。

## 非目标

- 不实现可滚动面板或历史输出重排。
- 不持久化输出展开偏好。
- 不改变工具执行、错误处理或输出脱敏逻辑。

## 验收标准

- 默认长工具输出仍折叠，并提示可用 `Ctrl+O` 展开后续输出。
- 触发 `Ctrl+O` 后，后续长工具输出完整打印，不包含折叠的 hidden-lines 提示。
- 再次触发 `Ctrl+O` 后恢复折叠。
- `Ctrl+O` 不提交输入、不清空输入缓冲。
- 单元测试覆盖直接 handler 与 raw stdin dispatch。
