# Dream Duplicate Guard Spec

## 背景

`/dream` 已提供显式本地记忆沉淀入口，但当前实现只要启发式抽取到候选就直接写入 MemoryStore。用户连续执行 `/dream` 时，完全相同的记忆会重复进入本地长期记忆，降低检索质量，也让本地数据留存变脏。

## 目标

- `/dream` 写入前跳过已存在的完全相同记忆内容。
- 所有候选均已存在时，不调用 `saveToDisk()`，并提示没有新的本地记忆。
- 部分候选已存在时，只写入新增候选，输出新增数量。
- 去重只依赖本地 MemoryStore 已导出的 persisted entries，不读取 session 文件、不调用模型、不联网。

## 非目标

- 不做语义去重或相似度去重。
- 不迁移、删除或合并已有重复记忆。
- 不改变 AutoDream 后台逻辑。

## 验收标准

- 单元测试覆盖 `/dream` 全重复时不写入、不保存。
- 单元测试覆盖 `/dream` 部分重复时只写入新增项。
- 输出仍不泄漏候选正文。
