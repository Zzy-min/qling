# `qling` TUI 双 Ctrl+C 退出规格（2026-06-04）

## 背景

TUI 当前的 `Ctrl+C` 只清空输入缓冲区。若输入为空，用户按 `Ctrl+C` 只能看到 `^C`，但没有明确退出路径；这会让终端交互显得迟钝，也不符合 Claude Code 类 CLI 的肌肉记忆。

## 目标

- 输入缓冲区非空时，`Ctrl+C` 只清空当前输入，不退出。
- 输入缓冲区为空时，第一次 `Ctrl+C` 给出本地提示，不调用模型、不联网、不写磁盘。
- 在短时间窗口内第二次 `Ctrl+C` 触发已有退出路径，等价提交 `exit`。
- 超出窗口后再次 `Ctrl+C` 重新作为第一次提示处理。
- 行为只存在于当前 TUI 进程内存，不新增持久状态。

## 非目标

- 不实现运行中 AgentLoop 的强制取消。
- 不修改 classic REPL。
- 不改变 `exit/q/quit` 命令语义。

## 行为

- 双击窗口为 2 秒。
- 第一次空输入 `Ctrl+C` 输出：`再次 Ctrl+C 退出，或输入 exit`。
- 第二次空输入 `Ctrl+C` 调用输入回调提交 `exit`。
- 非空输入 `Ctrl+C` 清空输入并重置双击状态。

## 验收

- 单测覆盖非空输入 `Ctrl+C` 只清空不退出。
- 单测覆盖空输入第一次 `Ctrl+C` 不退出并输出提示。
- 单测覆盖 2 秒内第二次空输入 `Ctrl+C` 提交 `exit`。
- 单测覆盖超时后第二次 `Ctrl+C` 不退出。
- `npm run build`、目标单测、`npm run ci:check` 通过。
