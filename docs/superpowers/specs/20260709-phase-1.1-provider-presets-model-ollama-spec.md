# Spec: Phase 1.1 — Provider Presets / 会话切换 / Ollama 探测

**日期**: 2026-07-09  
**状态**: Accepted  
**版本**: 落在 `1.0.0` 之后的功能增量（不强制升到 1.1.0，除非另行决定）  
**父路线**: `20260709-agent-cli-competitive-analysis-and-v1-roadmap-spec.md` Phase 1.1

---

## 1. 问题

1. Provider 预设只存在于 `cli/setup.ts` 私有表，命令与 runtime 无法复用。  
2. `/model` 仅改 `model` 字符串，不能切换 `provider` / `endpoint`。  
3. `doctor` 不探测本机 Ollama，本地模型路径不「一等公民」。  
4. 本地 Ollama 常不需要 API key，但 AgentLoop 构造强制 key（体验摩擦）。

---

## 2. 目标

| ID | 目标 | 验收 |
|----|------|------|
| P1 | 单一真相源 `providers/presets` | setup 与 `/model` 同源 |
| P2 | `/model` 可查看完整 LLM 会话态 | 显示 provider/endpoint/model |
| P3 | `/model list` 列出预设 | 含 id、display、model、endpoint |
| P4 | `/model use <preset\|id>` 切换整包 LLM 配置 | 进程内生效；默认不写盘 |
| P5 | `/model <name>` 兼容只改 model | 保持旧行为 |
| P6 | `doctor` 探测 Ollama | `http://127.0.0.1:11434` 可达性 check + 建议 |
| P7 | 本地 loopback endpoint 允许空 API key | 构造不因缺 key 失败 |

### 非目标

- 不实现完整 75+ provider 目录  
- 不默认把 `/model` 写入 `~/.qling/.env`（可选 `--save` 可后续）  
- 不实现多模型并行路由 / fallback 链  
- 不拉起 Ollama 进程或自动 `ollama pull`

## 3. 设计

### 3.1 ProviderPreset

```ts
export interface ProviderPreset {
  id: string;           // deepseek | ollama | openai | ...
  displayName: string;  // 中文/展示名
  provider: string;     // QLING_LLM_PROVIDER 值
  endpoint: string;
  model: string;
  keyHint?: string;
  requiresApiKey: boolean;
  tags?: string[];      // e.g. ["local","china"]
}
```

预设至少覆盖现有 setup 1–10（含 Ollama / OpenAI / 国内主流）。

### 3.2 解析 API

- `listProviderPresets(): ProviderPreset[]`
- `getProviderPreset(idOrAlias: string): ProviderPreset | undefined`
- `resolveModelCandidates(): string[]` — 各 preset 默认 model 去重

### 3.3 AgentLoop 会话 LLM 态

新增：

- `getProvider()` / `getEndpoint()`
- `applyLlmSession({ model?, provider?, endpoint?, apiKey? })`  
  同步更新 `this.config` 与相关 env（`QLING_LLM_*`），重建依赖 model 的 compactor。

空 key 规则：当 `endpoint` 为 loopback（localhost/127.0.0.1/::1）或 preset `requiresApiKey===false` 时，允许 `apiKey` 为空并用占位 `local`。

### 3.4 `/model` 命令语义

| 输入 | 行为 |
|------|------|
| `/model` | 显示当前 provider / endpoint / model；提示 list/use |
| `/model list` | 列出预设表 |
| `/model use <id>` | 应用预设（provider+endpoint+model） |
| `/model <modelName>` | 仅切换 model（兼容） |

边界：默认仅当前进程；明确不写配置文件。

### 3.5 doctor Ollama check

- `id: "ollama"`
- 探测 `env.QLING_OLLAMA_URL` 或默认 `http://127.0.0.1:11434/api/tags`
- 超时 ≤ 800ms，仅 loopback  
- 可达 → pass；不可达 → warn + 推荐「安装 Ollama 或 `/model use ollama`」

允许通过 `DoctorOptions.ollamaProbe` 注入，便于单测。
