# G5.1 `qling run --json` 结构化事件输出规格

## 目标

为脚本、CI 和外部编排器提供稳定的无交互输出，同时保留轻灵已有的本地优先执行链与恢复证据链。

## 命令契约

- `qling run "任务" --json` 启用结构化输出；`--json` 可放在 `run` 前后。
- `--json` 只允许用于单次 `run` 模式。用于 `chat`、`repl` 或管理命令时返回 `CLI_INVALID_MODE_COMBINATION`，退出码为 `2`。
- stdout 使用 NDJSON：每行一个完整 JSON 对象，不输出 ANSI 或人类可读面板。
- 运行期诊断和兼容警告继续写入 stderr，不进入 stdout 数据流。
- 无交互 console 遇到 `ask` 审批时安全自动拒绝，不打印交互提示、不永久等待；拒绝结果进入后续执行事件与工具失败上下文。

## 事件模型

所有事件都包含：

- `schemaVersion: 1`
- `type: string`
- `timestamp: string`（ISO 8601）

执行事件从现有本地 `ExecutionEventBus` 转发，包含现有的 `runId`、`sessionId`、`status`、`tool`、`stage` 等非敏感字段。最终事件为：

```json
{"schemaVersion":1,"type":"result","timestamp":"...","ok":true,"mode":"run","result":"...","session":{"id":"...","turnCount":1},"usage":{"totalTokens":0,"promptTokens":0,"completionTokens":0,"source":"provider"}}
```

失败事件为：

```json
{"schemaVersion":1,"type":"error","timestamp":"...","ok":false,"mode":"run","error":{"code":"RUN_FAILED","message":"..."}}
```

## 失败与退出码

- 参数错误：结构化 `error` 事件，退出码 `2`。
- 配置、通道初始化或运行错误：结构化 `error` 事件，退出码 `1`。
- 成功：至少一个终态 `result` 事件，退出码 `0`。
- JSON 序列化必须通过单一 formatter；不把 API key、完整配置或个人路径新增到事件中。

## 兼容性

- 未传 `--json` 时保持当前文本输出和错误面板。
- v1 只新增字段，不改变既有字段语义；未来破坏性变更必须提升 `schemaVersion`。
