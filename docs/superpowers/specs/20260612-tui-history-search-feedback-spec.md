# TUI History Search Feedback Spec

## 背景

qling TUI 支持 `Ctrl+R` 在本地输入历史中搜索，但当前未命中时只重绘输入栏，没有任何反馈。用户无法区分“没有匹配历史”“历史为空”或“快捷键未生效”，影响 terminal-native 交互的确定性。

## 目标

- `Ctrl+R` 命中历史时保持现有行为。
- `Ctrl+R` 未命中时显示一条本地轻量提示。
- 未命中不得清空或修改当前输入草稿。
- 未命中不得提交输入、不得调用模型、不得写入历史。

## 非目标

- 不改变 `InputBuffer` 的持久化格式。
- 不实现交互式增量搜索 UI。
- 不改变 `↑` / `↓` 历史导航语义。

## 验收标准

- 单元测试证明 `Ctrl+R` 未命中时输出“无匹配历史”类提示。
- 单元测试证明未命中后输入值和光标位置保持不变。
- 单元测试证明未命中不会触发提交。
- 定向测试、完整 CI、旧名扫描、audit、diff 检查通过。
