# `qling` TUI 历史检索规格（2026-05-31）

## 背景

为了提升长会话交互效率，TUI 已支持多行输入和历史上下切换，但用户仍需要反复按上方向键才能找回较早命令。Claude Code 类交互里，历史检索应尽量轻量、可预测，并且不依赖远端或持久化索引。

## 目标

- 新增 `Ctrl+R` 历史检索快捷键。
- 当输入区非空时，用当前输入内容作为关键字，恢复最近一条包含该关键字的历史命令。
- 当输入区为空时，`Ctrl+R` 等价于向上选择最近历史，降低空操作概率。
- 检索只读取当前 TUI 进程内历史，不写入磁盘，不上传内容。
- 检索命中后光标移动到命令末尾，方便继续编辑或直接提交。

## 非目标

- 不实现交互式反向搜索 UI。
- 不做跨会话持久化历史。
- 不做模糊匹配、排序权重或知识库检索。
- 不改变 `Enter`、`Ctrl+N`、上下方向键的现有行为。

## 验收

- 单测覆盖非空关键字命中最近历史、空输入回退到最近历史、未命中时不修改输入。
- TUI header 展示 `Ctrl+R 搜索历史`。
- `chat` smoke 仍可用 `exit\n` 正常退出。
- `npm run build`、局部测试和 `npm run ci:check` 通过。
