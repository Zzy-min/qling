# 未知 Slash 命令候选建议规格（2026-06-01）

## 背景

轻灵的 slash 命令数量持续增加。当前用户输错命令时只看到“未知指令，输入 /help”，需要再打开长帮助列表自行查找。为了提升接近 Claude Code CLI 的顺滑交互，未知命令应直接给出本地最可能候选。

## 目标

- 未知 slash 命令输出本地候选建议，例如 `/expors` 建议 `/exports`。
- 中文别名也参与匹配，例如 `/导出列` 建议 `/导出列表`。
- 候选来源只来自已注册的 `COMMANDS` 的 `name` 与 `aliases`。
- 输出包含下一步提示：可直接尝试候选命令，或使用 `/help <topic>` 查看聚焦帮助。
- 无可靠候选时保留简洁通用提示，不输出误导性建议。

## 非目标

- 不引入模型纠错。
- 不联网、不读取状态文件、不读取会话正文。
- 不自动执行候选命令。
- 不改变已知命令的调度行为。

## 行为

- `/expors` 返回未知命令，并建议 `/exports` 与 `/help exports`。
- `/导出列` 返回未知命令，并建议 `/导出列表` 与 `/help 导出列表`。
- `/zzzzzz` 返回未知命令和 `/help` 通用提示，不给候选。
- 匹配算法需稳定、可测试、无随机性。

## 验收

- 单元测试覆盖英文 typo、中文别名 typo、无候选降级。
- 现有 slash 命令测试继续通过。
- `npm run build`、目标测试和 `npm run ci:check` 通过。
