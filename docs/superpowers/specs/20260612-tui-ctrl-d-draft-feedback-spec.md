# TUI Ctrl+D Draft Feedback Spec

## 背景

qling TUI 目前在空输入时 `Ctrl+D` 会退出；在非空输入时为了避免误删草稿，会直接忽略。但静默忽略会让用户无法判断是快捷键失效、终端没收到按键，还是 qling 有意保护草稿。

## 目标

- 非空输入时按 `Ctrl+D` 显示一条本地提示。
- 非空输入时不得提交、不得退出、不得清空或修改草稿。
- 空输入时 `Ctrl+D` 退出行为保持不变。
- 不调用模型、不写入输入历史、不改变持久化格式。

## 非目标

- 不改变 `Ctrl+C` 渐进退出语义。
- 不新增确认对话或交互模式。
- 不改变 slash exit aliases。

## 验收标准

- 单元测试证明非空输入按 `Ctrl+D` 会输出提示。
- 单元测试证明草稿内容和光标位置保持不变。
- 单元测试证明不会提交 `exit` 或当前草稿。
- 定向测试、完整 CI、旧名扫描、audit、diff 检查通过。
