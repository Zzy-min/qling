# `qingling` TUI 多行输入规格（2026-05-31）

## 背景

为了贴近 Claude Code 的交互体验，长任务提示、计划、错误日志和多段上下文需要能在输入区自然换行。当前 TUI 只有单行输入，长文本不易编辑。

## 目标

- 保持 `Enter` 提交不变，避免破坏现有终端和 smoke 测试。
- 新增 `Ctrl+N` 插入换行，支持多行 prompt。
- 左右移动、退格、历史切换仍可稳定工作。
- 多行输入提交后保留内部换行，发送给 AgentLoop。

## 非目标

- 不实现全屏编辑器。
- 不改变 paste 行为。
- 不改变 `Ctrl+C` 清空输入行为。

## 验收

- 单测覆盖输入缓冲插入字符、插入换行、跨行退格、左右移动、历史恢复。
- `chat` smoke 仍可用 `exit\n` 正常退出。
- `npm run build` 和 `npm run ci:check` 通过。
