# `qling` 本地导出索引规格（2026-05-31）

## 背景

`/export` 已能把当前会话写入本地 Markdown，但用户仍需要记住文件目录或手动查找。为了让“数据留存本地”可发现、可核对，需要提供一个只读索引命令列出最近导出的文件。

## 目标

- 新增 `/exports [count]` slash command，中文别名 `/导出列表 [count]`。
- 列出 `<stateDir>/exports/` 下的 Markdown 导出文件。
- 默认显示最近 10 条，最多显示 50 条。
- 按文件修改时间倒序展示。
- 输出文件名、修改时间、大小和绝对路径。
- 命令只读取文件元数据，不读取文件正文、不调用模型、不联网。

## 非目标

- 不删除、移动或覆盖导出文件。
- 不打开编辑器或系统文件管理器。
- 不做正文搜索、摘要或云端同步。
- 不改变 `/export` 的 Markdown 格式和写入路径。

## 行为

- `/exports` 显示最近 10 条导出记录。
- `/exports 20` 显示最近 20 条。
- `/导出列表` 与 `/exports` 等价。
- `count` 非法、小于等于 0 或缺省时使用 10；超过 50 时截断为 50。
- 导出目录不存在或没有 `.md` 文件时输出空态提示，并建议先使用 `/export`。

## 验收

- 单测覆盖缺失目录、空目录、排序、count 规则和格式化输出。
- slash 单测覆盖 `/help`、`/exports` 和 `/导出列表`。
- 输出不包含 Markdown 文件正文内容。
- `npm run build`、相关单测和 `npm run ci:check` 通过。
