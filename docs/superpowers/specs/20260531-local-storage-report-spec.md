# `qling` 本地存储盘点规格（2026-05-31）

## 背景

当前 `/privacy` 能说明本地数据边界，`/context` 能展示上下文统计，`/exports` 能列出导出文件。但用户仍缺少一个直接入口，快速知道轻灵在本机留下了哪些目录、文件数量和大致占用。

## 目标

- 新增 `/storage` slash command，中文别名 `/存储`。
- 只读盘点本地 state、sessions、exports、cache 目录。
- 输出 workspace、各本地目录路径、文件数量、目录数量、占用大小和是否截断。
- 缺失目录不视为错误，显示 missing。
- 默认最多扫描 5000 个目录项，避免大型 cache 拖慢交互。
- 命令只读取文件元数据，不读取文件正文、不调用模型、不联网。

## 非目标

- 不清理、删除、移动或压缩任何文件。
- 不扫描整个 workspace。
- 不读取会话正文、导出正文或 cache 文件内容。
- 不上传诊断数据，不改变 `/privacy`、`/context`、`/exports` 行为。

## 行为

- `/storage` 显示本机轻灵存储盘点。
- `/存储` 与 `/storage` 等价。
- 目录存在时递归统计文件、目录和字节数。
- 目录不存在时显示 `missing`，文件数和大小为 0。
- 触达扫描上限时显示 `truncated`，提示结果是近似盘点。

## 验收

- 单测覆盖缺失目录、正常统计、扫描上限截断、格式化输出不包含文件正文。
- slash 单测覆盖 `/help`、`/storage` 和 `/存储`。
- `npm run build`、相关单测和 `npm run ci:check` 通过。
