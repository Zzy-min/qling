## 1. 目标

| ID | 目标 | 验收 |
|----|------|------|
| S1 | write/patch 默认仅允许工作区 | `QLING_WRITE_SANDBOX=workspace` 默认 |
| S2 | 敏感文件写拦截（.env / 密钥） | 单测 deny |
| S3 | 网络策略模式 strict/open/deny | env 可配 + url_fetch/browser_fetch |
| S4 | browser_fetch 走同一 Guard | 私网/前缀被拦 |

| S6 | Docker 运行文档 | `docs/docker.md` |

### 非目标

- 完整 OS 级 sandbox / seccomp  
- 强制容器默认运行

## 2. 设计

### 2.1 写沙箱 `QLING_WRITE_SANDBOX`

| 值 | 行为 |
|----|------|
| `workspace`（默认） | write/patch 仅工作区（无 workspace 时用 cwd） |
| `roots` | 兼容旧行为：workspace + state + cache |
| `off` | 关闭路径边界（仅排障） |

### 2.2 敏感写路径

默认拒绝写入：`.env*`、`*.pem`、`*.key`、`id_rsa`、`credentials.json` 等。  
`QLING_ALLOW_SENSITIVE_WRITE=1` 可显式放开。

### 2.3 网络 `QLING_GUARD_NETWORK_MODE`

| 值 | 行为 |
|----|------|
| `strict`（默认） | 沿用 allowed_url_prefixes（默认仅 https://）+ 拒私网 |
| `open` | 允许 http/https，仍拒私网 |
| `deny` | 拒绝所有 url_fetch/browser_fetch |

### 2.4 Docker

文档说明：挂载工作区、注入 env、不把密钥打进镜像。  
可选示例 `Dockerfile` / `docker-compose.yml` 放在 `packaging/docker/`。
