# `qingling` TUI 输入队列背压规格（2026-06-01）

## 背景

TUI 已支持执行期间继续输入，并通过本地内存队列串行处理输入。当前队列没有上限；如果用户误粘贴大量内容、终端重复提交或脚本持续写入 stdin，待处理输入可能无限增长，带来内存增长和后续执行不可预测的问题。

## 目标

- 为本地输入队列增加待处理数量上限。
- 上限只统计等待处理的输入，不包含正在运行的当前输入。
- 队列满时拒绝新增输入，并输出不含用户输入正文的提示。
- 拒绝不写磁盘、不联网、不调用模型，不进入 `AgentLoop`。
- 默认 TUI 上限为 20 条待处理输入，保护交互稳定性。
- 保持正常输入、排队通知、串行顺序和错误不中断行为不变。

## 非目标

- 不持久化队列。
- 不实现队列编辑、重排、取消命令。
- 不自动总结或压缩被拒绝输入。
- 不在拒绝提示里展示用户输入正文。

## 行为

- `SerialInputQueue` 支持 `maxPending` 选项。
- `maxPending` 缺省为无限制，用于保持模块复用兼容性。
- `StreamingREPL` 使用 `maxPending: 20`。
- 当 `pendingCount >= maxPending` 时，新增输入不入队，`enqueue()` 返回 `false`。
- 拒绝事件只包含 `pendingCount` 和 `maxPending`，不包含输入正文。
- REPL 输出 `输入队列已满，已忽略本次输入: pending/max`。

## 验收

- 单测覆盖达到 `maxPending` 后拒绝新增输入。
- 单测覆盖被拒绝输入的 handler 不会执行。
- 单测覆盖 `onRejected` 事件不包含输入正文。
- `StreamingREPL` 接入拒绝提示，文案不包含用户输入正文。
- `npm run build`、目标单测和 `npm run ci:check` 通过。
