## G3.3 权限流水线

顺序（文档化 + 代码一致）:

1. **Plan Mode** 硬拒绝 / 计划目录写白名单  
2. **Permission rules**（矩阵 + 默认 mode）  
3. **Remembered grant**（本会话用户已 allow 的工具）  
4. **Rate limit / classifier**  
5. **Custom PreToolUse hooks**  
6. **Mode**（normal=ask / auto=allow 默认，经 setPermissionMode）

交付:

- `src/guard/permission-grants.ts`
- hooks 在 ask 前查 grant
- approval allow 时 remember
- `/permissions pipeline|grants|forget`

## G3.4 Sandbox profiles

| Profile | 写 | bash cwd | 敏感文件 |
|---------|----|----------|----------|
| workspace | 仅工作区 | 工作区优先 | 默认拦 |
| read-only | 全禁 | 可读 cwd | — |
| strict | 仅工作区 | **必须**工作区 | 始终拦 |
| roots | workspace+state+cache | roots | 默认拦 |
| off | 不限 | 不限 | 可 env 放行 |

- `src/runtime/sandbox-profile.ts`
- 接线 write/patch/bash
- `/sandbox [status|workspace|read-only|strict|roots|off]`
- env: `QLING_SANDBOX_PROFILE`（兼容 `QLING_WRITE_SANDBOX`）
