# G3.2 Background tool 统一 — 实施计划

**状态**: 已完成（2026-07-16）  
**对标**: Grok bg shell · task_id · list / wait / kill · TUI 一行通知  
**基线**: 已有 session loop tasks、mission、同步 bash；缺统一 `task_id` 与可后台 shell。

## 目标

1. 进程内 `BackgroundTaskRegistry`：统一 `task_id`（`bg_…`）
2. `bash` 支持 `background: true` → 立即返回 `task_id`，进程继续跑
3. 用户侧：`/tasks` 列出 shell bg + loop；`wait` / `kill` / `cancel`
4. Agent 侧：`bg_wait` / `bg_kill` 工具（只读 list 走 `/tasks` 或 `bg_list`）
5. TUI：start / done / fail / killed 一行通知
6. 不重做 mission：仅可选聚合展示；loop 任务保留原 id 体系

## 非目标（本轮）

- OS 硬沙箱（G3.4）
- 跨进程 durable bg（daemon 已有 loop）
- 全屏 Ctrl+G 舰队（G4）

## 验收

- `background: true` 的 bash 不阻塞工具返回，含 `task_id`
- `/tasks` 可见 running/completed
- `kill` 终止进程树（Windows taskkill / Unix SIGTERM→SIGKILL）
- `wait` 等到终态或超时
- Plan mode 仍禁 bash（含 background）
- 单测绿 + `npm run build`
