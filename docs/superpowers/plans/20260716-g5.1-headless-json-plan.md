# G5.1 `qling run --json` 实施计划

1. 在 CLI 参数解析层记录 `outputFormat`，限制 `--json` 只用于 `run`，并更新帮助文本。
2. 新增无副作用的 NDJSON formatter，覆盖执行事件、成功结果、失败结果和会话 usage 摘要。
3. 在 `run` 路径订阅现有 `ExecutionEventBus`，逐行输出证据事件；成功或失败时输出终态事件。
4. 对参数、配置、通道和运行失败统一选择文本或 JSON 错误输出，并保持退出码契约。
5. 增加 parser/formatter 单元测试和真实本地 HTTP provider CLI 冒烟测试。
6. 运行定向测试、构建、全量单测和差异检查；失败时优先修复受本改动影响的回归。
