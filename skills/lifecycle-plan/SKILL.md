# 生命周期 · 计划（Plan）

## 步骤

1. 重述 Spec 中的验收标准（不得改写成功定义）。
2. 拆成 **原子任务**（每步可独立验证，通常 15–45 分钟量级）。
3. 标注依赖顺序与可并行只读探索。
4. 标明将触碰的文件路径（估计即可）。
5. 写出 **测试计划**：先写什么测、如何证明完成。
6. 风险与回滚点。

## 输出格式

```markdown
## Plan
1. [ ] 任务… — 文件… — 验证…
2. [ ] …
## 风险
- …
```

## 不要

- 不要跳过测试步骤。
- 不要一次计划「重写整个仓库」。
