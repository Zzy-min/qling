# Add Function

## 何时使用

用户要求「加一个函数」「导出 xxx」「补一个工具方法」时加载。

## 步骤

1. **读现有模块** — 风格、命名、错误处理、导入习惯
2. **设计签名** — 参数/返回值/边界；与相邻 API 一致
3. **实现** — `patch` 或 `write` 最小增量
4. **调用点** — 若用户给了使用场景，同步更新调用处
5. **测试** — 能测则加/改测试；不能测则给手动验证步骤

## 约束

- 不引入新依赖，除非用户同意
- 不顺手大重构
- 公开 API 变更要在回复里写清楚

## 验收

- [ ] 新符号可 import / 可调用
- [ ] 有验证命令或测试
